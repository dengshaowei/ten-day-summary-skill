# 十天总结 (Ten Day Summary)

## 概述

这个Skill帮助用户将10天的零散日常记录整理成结构化、美观的总结文档。适用于周期性个人回顾和复盘。

## 工作流程

### 第一步：确认时间范围
询问用户这是第几个10天，以及日期范围（例如：第13个10天，5月1日-5月10日）。

### 第二步：获取原始记录
询问用户原始记录的位置：
- 直接发送给我
- 需要我去聊天记录里提取
- 在某个文档/笔记里
- 口述，我来整理

### 第三步：整理格式
按照以下标准格式整理：

**标题格式：**
```
**第X个10天【开始日期-结束日期】**
```

**每日格式：**
```
**日期（如：5月1日）**

【运动】：记录内容

【社交】：记录内容

【生活】：记录内容

【学习】：记录内容

【工作】：记录内容

【思考】：记录内容（如有）

【自我察觉】：记录内容（如有）

【成果】：记录内容（如有）

【收获】：记录内容（如有）

【关系】：记录内容（如有）

【技能探索】：记录内容（如有）

【认知】：记录内容（如有）

---
（重复每一天...）
```

**10天总结格式（可选）：**
```
## 10天总结：PDCA 计划执行检查处理
1. **关键发现/成果：简要描述
2. **关键发现/成果：简要描述
3. ...
```

### 第四步：输出预览
整理完成后，先展示给用户确认，确认无误后再写入文档（如果需要）。

## 分类标签说明

**核心标签（每日常用）：**
- 【运动】健身、运动相关记录
- 【社交】与人交流、聚会、活动
- 【生活】日常琐事、饮食、休息、体重等
- 【学习】阅读、听课、背单词等学习内容
- 【工作】工作任务、项目进展

**扩展标签（按需使用）：**
- 【思考】个人感悟、深度思考
- 【自我察觉】对自身状态、情绪的觉察
- 【成果】完成的事项、产出物
- 【收获】获得的知识、经验、礼物等
- 【关系】亲密关系、家庭关系的维护
- 【技能探索】新技能学习、工具探索
- 【认知】观念更新、认知升级

## 格式规范

1. **标题层级**：使用 `**加粗**` 而非 `# 标题`
2. **日期分隔**：每天之间用 `---` 分隔
3. **空行**：每个标签内容后保留空行，确保可读性
4. **多行内容**：同一标签有多项内容时，用 `+` 或换行分隔
5. **高亮**：关键感悟、金句或重要发现用 `**加粗**` 突出
6. **标题合并**：日期范围直接跟在"第X个10天"后面，如 `**第12个10天【4月21日-4月30日】**`

## 使用提示

- 如果用户提供了不完整的记录，根据上下文合理补全或标注"（待补充）"
- 保持用户的原始语气和表达方式，只做格式整理
- 对于零散的时间记录，按天归类；无法确定日期的，归入"【其他】"
- 如果某类标签某天没有内容，可以省略该标签
